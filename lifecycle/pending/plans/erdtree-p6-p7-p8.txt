## 0. CONTEXT & GOALS

Problem statement
  P0–P5 are COMPLETE and verified (see lifecycle/archive/audits/erdtree-v0.1/
  FINAL.md): the safety/awareness core, the three high-leverage tools
  (services, packages, logs), the Ollama client + prompt assembly, the closed
  agentic loop + tool-call benchmark, and the product login shell with the
  dead-man bash fallback. The frozen tool interface lives in
  core/tools/__init__.py and the loop is driven by core/agent/repl.py.

  Three capabilities remain before the v0.1 framework is feature-complete
  (installer/tier-plumbing/training are separate later phases):
    P6  the seven remaining tools (network, firewall, users, disk, processes,
        hardware, files) — the lockout/data-loss blast-radius tools.
    P7  retrieval (RAG) exposed as a tool the loop calls only when it decides
        it needs facts — local corpus, local embeddings, local vector index.
    P8  invisible memory: rolling compaction, a per-host facts preamble, and
        episodic RAG-over-the-audit-log, so the session never shows a context
        limit ("the amnesia moment" is the cardinal UX sin).

Success criteria (measurable)
  SC-P6.1  All 7 tools implement the FROZEN interface (ToolSpec/OpSpec/ArgSpec/
           ToolResult) in core/tools/__init__.py, self-register into the
           module-level `registry` singleton, and shell out ONLY via
           run_subprocess (no tool-specific Python libs — I1/portability).
  SC-P6.2  Every lockout/data-loss op routes through the EXISTING hardened
           classifier (core/agent/permissions.classify) as DESTRUCTIVE and is
           gated CONFIRM_TYPED (typed "DESTROY"), never auto-confirmed, never
           run non-interactively. VERIFIED by feeding each tool's synthesized
           command line through classify() in tests.
  SC-P6.3  synthesize_command() in core/agent/repl.py renders a FAITHFUL shell
           command line for every new (tool, op) pair, so the classifier sees
           the real blast radius (e.g. disk.format -> "mkfs.ext4 /dev/sdb1",
           firewall.panic -> "firewall-cmd --panic-on"). No new (tool, op) may
           fall through to the bare `f"{tool} {op}"` default-WRITE floor when a
           more severe class is warranted.
  SC-P6.4  main.py registers all 7 new tools (import side-effect), and the
           full registered toolset advertises cleanly through router schemas
           (registry_schemas) with zero I2-forbidden terms in any description.
  SC-P7.1  rag/ builds a corpus + local embeddings + a local vector index with
           ZERO network calls (offline-built; I1). Index footprint within the
           plan's budget (~3–5 GB SSD, ~1–2 GB RAM at query time).
  SC-P7.2  Retrieval is a TOOL ("docs") implementing the frozen interface; a
           factual query returns ~3 tight reranked chunks; a routine op
           ("restart nginx") triggers ZERO retrieval (the loop chooses to call
           it, it is never prepended).
  SC-P7.3  The retrieval engine is a REUSABLE component (rag/retrieve.py) that
           P8 points at the audit log without a second retriever being built.
  SC-P7.4  Per-tier retrieval budget (k chunks, max chars) is a config knob
           read from outside core/ (I6); core/tools/docs.py never hardcodes a
           tier name.
  SC-P8.1  Rolling compaction keeps tool-call OUTCOMES (exit code + summary)
           and drops verbose raw stdout/stderr once reasoned over; recent turns
           stay verbatim (deixis "restart it" / "the one we just did" resolves).
  SC-P8.2  A per-host facts file is injected as a tiny preamble every turn.
  SC-P8.3  Episodic memory: a fact established many tasks ago is recalled via
           audit-log retrieval and answered AS IF KNOWN — no "out of context",
           no reset, no re-ask. The compaction threshold is a per-tier knob.
  SC-P8.4  NO user-visible context-limit / reset / "amnesia" language EVER
           (I2 + UX). A test corpus probes specifically for leakage.

Non-goals (explicitly OUT of scope here)

Assumptions made (call out if any is wrong — it reshapes a phase)

## 1. ARCHITECTURE OVERVIEW

How P6/P7/P8 slot into the EXISTING, BUILT loop (core/agent/repl.py):

  user English -> Repl.run_turn
     -> TurnContext.snapshot_text()           [P8 injects facts preamble here]
     -> assemble(...) prompt                   [P8 supplies compacted history]
     -> responder (Ollama)                     [UNCHANGED]
     -> Router.route -> ParsedCall(s)          [UNCHANGED; new tools auto-advertise]
     -> for each call:
          synthesize_command(call)             [P6 EXTENDS this — keystone]
          perm.classify(command, exec_ctx)     [UNCHANGED hardened classifier]
          _resolve_gate (ALLOW/CONFIRM/TYPED/REFUSE)   [UNCHANGED]
          registry.dispatch -> tool.execute    [P6 adds 7 tools; P7 adds docs]
          audit.write(...)                     [UNCHANGED; P8 reads this log later]
          context.invalidate() on mutation     [UNCHANGED]
     -> English answer terminates the turn

The three phases touch THREE distinct seams:

  P6  the TOOL seam (core/tools/*) + the synthesize_command() classifier bridge
      in repl.py + the tool-import list in main.py. NO change to the interface,
      the router, the permissions module, or the audit module.

  P7  a NEW package rag/ (corpus build, embed, index, retrieve) + ONE new tool
      core/tools/docs.py that wraps rag.retrieve behind the frozen interface.
      docs.retrieve is a READ op (no gate friction). main.py imports it.

  P8  a NEW module core/agent/memory.py (compaction + window management) that
      Repl consumes for `history`, + core/context/facts.py (preamble) that
      TurnContext.snapshot_text composes in, + an episodic retriever that
      REUSES rag/retrieve.py pointed at /var/log/{tier}/audit.jsonl. Repl.run_turn
      changes from `history=[]` (currently hardcoded, repl.py line ~237) to a
      memory-managed history; this is the one surgical edit to the built loop.

Invariants threaded through every phase (from CLAUDE.md + original §1):
  I1  No external API calls at runtime. Local Ollama + local index only. The
      corpus EMBED step is offline. docs.py opens no socket.
  I2  No AI/LLM/model/agent/agentic language in user-facing strings. NOTE the
      prompt.py forbidden set already bans "model", "agent", "inference",
      "ollama", "neural", "llm", "ai", "gpt", "machine learning". P7/P8 user-
      facing text and tool DESCRIPTIONS must clear that same filter.
  I3  Permission gate before every write/destructive op; destructive = typed
      "DESTROY", never auto-confirmed, never non-interactive. P6 satisfies this
      THROUGH the existing classifier — never by self-classifying in a tool.
  I4  Audit-log every operation (the REPL already does this for every dispatch,
      including refusals and misses). P6 tools need no audit code of their own.
  I5  System context always injected. P8's facts preamble augments, never
      replaces, the live snapshot.
  I6  No tier/product names in core/. P7 budget + P8 threshold + facts path are
      opaque config values, read like AppConfig already reads ERDTREE_*.
  I8  Performance is a feature. Retrieval is opt-in (not every turn). Compaction
      is cheap (string/token accounting, no model calls).
  I9  Dead-man fallback is upstream (P5) and untouched. P6/P7/P8 must never
      raise out of run_turn in a way that defeats it: every new failure path
      degrades to a ToolResult or a safe fallback string, never an exception
      that escapes the loop.

### PHASE 6: The Seven Remaining Tools + Classifier Bridge + Registry Wiring
            Goal: complete the toolset; every lockout/data-loss op is gated
                  DESTRUCTIVE through the EXISTING classifier.
=====================================================================
Prerequisites: none new (P2 froze the interface; P4 built the loop; P5 the
shell). All are COMPLETE.

Deliverables:
  - 7 tools (network/firewall/users/disk/processes/hardware/files), each a
    near-clone of the services.py STRUCTURE: per-op functions returning
    ToolResult via run_subprocess, a _DISPATCH table, a ToolSpec with per-op
    permission_class, self-registration into `registry`, the SELinux-hint
    helper where relevant.
  - The EXTENDED synthesize_command() in repl.py (the classifier bridge).
  - The 7 new tool imports in main.py.
  - Per-tool tests + the synthesize_command gate test.

Per-tool operation map + permission class (declared in each ToolSpec; the
REAL gate is the classifier via synthesize_command — the OpSpec class is the
advisory/pre-classify aid the router surfaces):

  network.py    (nmcli / ip)
    show/status (READ), connections (READ), interfaces (READ)
    bring_up    (WRITE)  ip link set <if> up / nmcli con up
    bring_down  (DESTR)  "drop the link you're on" -> typed; synthesize
                         "ip link set <if> down" (classifier: ip ... set = write
                         floor; ESCALATE in synthesize by emitting the down on a
                         link — see NOTE below)
    set_ip      (WRITE)  nmcli con modify / ip addr add
  firewall.py   (firewall-cmd)
    list (READ), get_zones (READ), query (READ)
    add_service/add_port/remove_service/remove_port (WRITE)
    reload (WRITE)
    panic_on (DESTR)  -> "firewall-cmd --panic-on" (classifier: DESTRUCTIVE)
    set_default_zone (WRITE; on a remote SSH host can lock out -> see NOTE)
  users.py      (useradd/usermod/passwd/userdel/gpasswd)
    list (READ), info (READ)
    add (WRITE), set_shell (WRITE), add_to_group (WRITE)
    lock (DESTR)     -> "usermod -L <user>"  (classifier: DESTRUCTIVE)
    delete (DESTR)   -> "userdel <user>"     (classifier: DESTRUCTIVE)
    remove_from_privgroup (DESTR) -> "gpasswd -d <user> wheel" (DESTRUCTIVE)
  disk.py       (lsblk/df/mkfs/parted/mount/umount/dd)
    usage (READ), list (READ), smart (READ via smartctl)
    mount (WRITE), unmount (WRITE)
    format (DESTR)    -> "mkfs.<fstype> <device>"     (classifier: DESTRUCTIVE)
    partition (DESTR) -> "parted <device> ..."         (classifier: DESTRUCTIVE)
    wipe (DESTR)      -> "wipefs -a <device>"           (classifier: DESTRUCTIVE)
    dd_write (DESTR)  -> "dd if=... of=<device>"        (classifier: DESTRUCTIVE)
  processes.py  (ps/top/kill/pkill/renice)
    list (READ), tree (READ), top (READ), info (READ)
    signal (WRITE)   -> "kill <pid>" (plain); BUT "kill -1 <pid>"-of-init or
                        kill-all -1 must synthesize the real argv so the
                        classifier's "kill ... -1" DESTRUCTIVE rule fires.
    renice (WRITE)
  hardware.py   (lscpu/lspci/lsusb/lsblk/free/sensors/dmidecode) — ALL READ
    cpu/memory/pci/usb/block/sensors/summary (all READ)
  files.py      (ls/stat/cat/find/cp/mv/rm/mkdir/chmod/chown)
    list/read/stat/find (READ)
    copy/move/mkdir/chmod/chown/write (WRITE)
    remove (WRITE normally; recursive/forced or system-path -> DESTRUCTIVE via
            the classifier when synthesize emits "rm -rf <path>")

Steps:
  1. (PARALLEL, 7 siblings) Implement each tool against the frozen interface,
     modeled on services.py. STRICT rules:
       * subprocess only via run_subprocess (no psutil, no pyroute2, no shutil
         for destructive ops — shell out so the classifier sees the command).
       * NO call to permissions or audit inside the tool (A3).
       * Every user-facing summary/description clears the I2 filter (reuse the
         prompt.py forbidden-term list as a test assertion).
       * SELinux-hint helper (copy services.py _maybe_selinux_hint) on tools
         where AVC denials are likely (files, disk, firewall, users).
       * Clamp/validate args (e.g. files.read line cap) like services.logs does.
  2. (SEQUENTIAL, after the 7 land) EXTEND synthesize_command() in repl.py:
     add one branch per new tool that renders the FAITHFUL command line so the
     classifier assigns the correct class. This is the load-bearing correctness
     step — see the dedicated gate in Validation. Conservatism rule: when a
     branch cannot render precisely, fall through to the existing default
     `f"{tool} {op}"` (WRITE floor) — NEVER emit a command that UNDER-states the
     blast radius. For destructive ops, emit the real dangerous form (mkfs.../
     userdel.../firewall-cmd --panic-on/ip link set <if> down) so the classifier
     ESCALATES to DESTRUCTIVE; the tool's own OpSpec class is advisory only.
  3. (SEQUENTIAL) Add the 7 imports to main.py so they self-register.
  4. Author tests: each test_tools_<x> mocks run_subprocess (dev host has no
     systemctl/nmcli/etc.), asserts structured results + that the op's
     synthesized command classifies to the EXPECTED gate.

Validation:
  - Per-tool: pytest test_tools_<x> green; read ops produce results without a
    gate; write/destructive ops produce results; I2 filter passes on every
    description + summary.
  - test_synthesize_command (THE GATE): for EVERY (tool, op) across all 7 tools,
    assert synthesize_command(ParsedCall(...)) -> classify(...) yields the
    intended OpClass/Gate. Specifically assert the lockout/data-loss set
    (firewall.panic_on, users.lock/delete/remove_from_privgroup, disk.format/
    partition/wipe/dd_write, network.bring_down, files.remove recursive) ALL
    classify DESTRUCTIVE -> CONFIRM_TYPED, and are REFUSE under a
    non-interactive ExecContext. This is SC-P6.2 + SC-P6.3 and is the phase's
    correctness keystone.
  - Registry: a one-shot test imports all tools + asserts
    registry.list_tools() contains the full 10 and registry_schemas() builds
    clean schemas for each.

Risks / Edge cases:
  - UNDER-GATING via a lazy synthesize_command branch (the cardinal P6 sin):
    e.g. emitting "disk format sdb1" instead of "mkfs.ext4 /dev/sdb1" would
    leave it at the WRITE floor. Mitigation: the test_synthesize_command gate
    is exhaustive over every (tool, op); route it to adversarial review.
  - A tool importing a Python lib that performs the op directly (bypassing the
    classifier, since the gate keys off the SYNTHESIZED string, not the tool's
    internals). Mitigation: hard rule + review — run_subprocess is the ONLY
    sanctioned execution path (it is literally the only one the classifier can
    reason about). A grep gate: no `import psutil|pyroute2|shutil.rmtree` etc.
  - Remote SSH lockout (firewall/network/users) — the classifier already has
    ExecContext.remote raising stakes; ensure synthesize passes through the
    interactive flag (Repl already builds ExecContext(interactive=...)). The
    confirm REASON text already names the lockout risk from the classifier.
  - hardware.py is all-READ — keep it trivial; do not over-engineer.

    inputs for tests; GPU on mossad for the full corpus). Pinned model.
  - rag/index.py — build + query a LOCAL vector index. DECISION: sqlite-vec
    (single-file, server-less, ISO-friendly, reusable by P8 episodic) unless
    the footprint measurement in Step 1 disqualifies it (then faiss). Record
    the verdict + measured SSD/RAM footprint in rag/index.py docstring +
    docs/decisions/0003-vector-index.md.
  - rag/retrieve.py — THE REUSABLE ENGINE. retrieve(query, k, max_chars) ->
    list[Chunk]: embed query -> ANN search (overfetch ~k*5) -> rerank
    (cross-encoder OR a cheap lexical+score rerank if no reranker ships) ->
    return ~k tight chunks within max_chars. Precision over recall. The engine
    takes the INDEX PATH as a parameter so P8 can point it at a different index
    (the audit-log episodic index) without code change (SC-P7.3).
  - core/tools/docs.py — the "docs" tool, frozen interface. ONE op: retrieve
    (READ — no gate friction). args: query (str, required), k (int, optional).
    execute() calls rag.retrieve and returns a ToolResult whose stdout is the
    joined chunks and summary is "Retrieved N reference passages." (I2-clean —
    NO "retrieval/embedding/model"; "reference passages" is operator language).
    Self-registers into `registry`. Reads k/max_chars defaults from the opaque
    config (ERDTREE_RETRIEVAL_K / ERDTREE_RETRIEVAL_MAXCHARS) — never a tier
    name (I6).
  - rag/fixtures/ — a tiny (~dozen-chunk) test corpus + prebuilt mini-index so
    test_rag_retrieve runs OFFLINE on the dev host with no GPU.

Steps:
  1. (FIRST, defines the contract) Choose the index backend by MEASURING the
     fixture-corpus footprint and the query latency; write the index format +
     the rag.retrieve(query, index_path, k, max_chars) -> [Chunk] signature.
     This signature is the contract the docs tool AND P8 episodic both bind to.
  2. (PARALLEL after Step 1)
     (a) corpus build + embed + index over the fixture corpus (full corpus is
         a DEFERRED-TO-MOSSAD background job; the recipe + license manifest
         ship now).
     (b) rag/retrieve.py + core/tools/docs.py against the Step-1 contract.
  3. Wire the docs tool into main.py (guarded import: if the index path is
     absent/unreadable, the tool registers in a degraded "index unavailable"
     mode that returns an empty-but-valid ToolResult — never crashes build_repl,
     I9). The loop simply gets no useful chunks and proceeds (it called docs by
     choice; an empty result is a valid answer).
  4. Track per-source licensing in rag/LICENSES.md: ship only redistributable
     sources in any ISO, OR ship the build recipe the user runs at firstboot.
     Default: SHIP THE RECIPE (the original §10 Q4 resolution leaning).

Validation:
  - test_rag_retrieve: against rag/fixtures, a factual query ("what does the X
    mount flag do") returns tight, relevant chunks within max_chars; an
    unrelated query returns low/zero results rather than noise. Index build over
    the fixture is fully offline (assert no socket opened — reuse the egress
    rig pattern, or assert via a no-network test shim).
  - test_tools_docs: the docs tool implements the interface, registers,
    classifies as READ (synthesize_command -> a read-shaped command, gate
    ALLOW), and degrades cleanly when the index is absent. I2 filter passes on
    description + summary.
  - SC-P7.2 behavioral check: a routine-op turn ("restart nginx") drives the
    loop WITHOUT a docs call (the model chose tools, not retrieval). Asserted
    in an e2e-style test with a scripted responder that does NOT emit a docs
    call — i.e. nothing prepends retrieval; it is purely opt-in.

Risks / Edge cases:
  - Over-retrieval burns prefill/KV-cache VRAM on 8 GB cards (original §3 P7
    warning). Mitigation: tight per-tier k/max_chars budget; precision rerank;

  - The reranker itself being a heavyweight model that violates the "no second
    big model" footprint. Mitigation: prefer a SMALL cross-encoder or a lexical
    rerank; the engine must run on CPU for tests. Decide in Step 1.
  - Corpus licensing (Arch wiki / SO). Mitigation: rag/LICENSES.md per-source;

  - docs.py accidentally synthesizing to a non-read command -> gate friction on

### PHASE 8: Invisible Memory — Compaction + Facts Preamble + Episodic RAG
            Goal: an immortal session that NEVER shows a context limit; old
                  facts recalled silently; deixis still resolves.
=====================================================================
Prerequisites: P4 loop (done), P1 audit log (done — the durable transcript),
P7 rag/retrieve.py (the reused engine). DEPENDS on P7.

Deliverables:
  - core/agent/memory.py — TranscriptMemory: owns the message-history window
    Repl currently hardcodes as `history=[]` (repl.py ~line 237). Provides:
      * record(assistant_msg, tool_result_msgs) — accumulate the turn.
      * compacted_history(threshold) — KEEP recent turns verbatim (for deixis);
        for older turns, KEEP tool-call OUTCOMES (the {exit_code, summary} the
        router already shapes in tool_result_message, router.py ~line 466) and
        DROP the verbose stdout/stderr once reasoned over. Threshold is the
        per-tier knob (chars/tokens). Pure stdlib accounting — no model calls.

        tasks without any visible reset.
  - core/context/facts.py — load a tiny curated per-host facts file (path from
    opaque config ERDTREE_FACTS_PATH) and expose it as a preamble string.
    TurnContext.snapshot_text() prepends it (I5 augmentation, not replacement).
    Absent file -> empty preamble (no error, no mention).
  - core/agent/episodic.py — EpisodicMemory: builds/refreshes a vector index
    over /var/log/{tier}/audit.jsonl (the audit path Repl already writes) USING
    rag/retrieve.py's engine pointed at that index (SC-P7.3 — no second

    prompt layer can inject so the loop answers "what did we do earlier" as
    KNOWING. The audit record fields (nl_input, translated_command, tool,
    result) are the episodic corpus (A5).
  - The surgical Repl edit: Repl.__init__ accepts optional `memory` +
    `episodic` + `facts`; run_turn uses memory.compacted_history() for the
    `history` arg and (when the loop's query is retrospective) lets the model
    call episodic recall via the SAME docs-tool mechanism OR a thin internal
    inject. PREFER: expose episodic recall through the docs tool's engine so
    the loop calls it like any other read — keeping the "loop decides" property
    and avoiding a special always-on path. Backward-compatible: memory=None
    preserves today's behavior exactly (existing tests stay green).

Steps:
  1. Implement TranscriptMemory with the keep-outcomes/drop-raw + recent-
     verbatim policy. Unit-test the policy directly: after N synthetic turns

Validation:

  - test_facts: facts preamble injected when present; empty + silent when absent.
